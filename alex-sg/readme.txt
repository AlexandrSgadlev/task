=== alex-sg ===
Задание делал на локальной машине, через Docker.


Структура проекта.
	/[language]/[Тип сделки]/[Тип недвижимости]/[Тип юнита]/

	Структура проекта была реализована при помощи кастомных типов записей. Для добавления страниц архивов по [Тип недвижимости], 
было решено сделать таксономию с терминами дублирующую ее. Идея заключается в следующем, пользователь добавляет термин в таксономию [Тип недвижимости],
и автоматически создается таксономия дублирующая ее. Тем самым можно не ограничиваться в количествах [Тип недвижимости], в отличие от [Тип сделки],
которые нужно добавлять кодом.
Также был добавлена автоматическая перезапись rewrite rules для этих таксономий синхронизация таксономий при сохранении постов. И конечно для SEO нужно задать переадресацию, так как создаются дублирующие страницы. 
	Что касается языка [language] то был использован плагин WP Multilang

Главная страница
	Тут все сделал примитивно через shortcode functions.php (Frone page shortcode). По переменным из шорткода можно управлять ими и менять их местами со странице.


Страницы каталога.
	Для сортировки по 2м полям, сначала выводим Hot, потом по дате добавления. Были добавлены мета поля, которые сохраняются автоматически.
И их сортировка functions.php (Sort).


Страница объекта.
	Для полей был использован плагин Advanced Custom Fields. Синхронизацию полей и внутренних данных, я не делал. Поля служат для визуальной составляющей. 
Также у меня не работает API с курсом валют. И не возвращает данные. И конвертировать площадь в метры квадратные я не стал, так как есть мета поля и к тому же использовать 
float переменные, как по мне это неправильно, потому что потеряется точность.
	Что касается блока Similar Property, он был сделан через WP_Query, functions.php (Add Similar Property). Для формы использовался плагин Contact Form 7.
